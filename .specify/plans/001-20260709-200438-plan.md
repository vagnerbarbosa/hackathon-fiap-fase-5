## Research.md (Phase 0)

### Decision: FastAPI Lifespan para Gerenciamento de Recursos

**Decision**: Usar `asynccontextmanager` com parâmetro `lifespan` do FastAPI para inicialização e cleanup de recursos (engine DB, conexões).

**Rationale**:
- Padrão recomendado pelo FastAPI para gerenciamento de ciclo de vida
- Permite setup async (conectar ao DB) e cleanup garantido
- Substitui os antigos `@app.on_event` (deprecated)

**Alternatives considered**:
- `@app.on_event("startup")` — deprecated, não recomendado
- Inicialização lazy — complexo, não garante ordem

### Decision: SQLAlchemy 2.0 Async com asyncpg

**Decision**: Usar `create_async_engine` com `postgresql+asyncpg` e `AsyncSession` para operações async.

**Rationale**:
- asyncpg é o driver asyncio mais maduro para PostgreSQL
- SQLAlchemy 2.0 tem suporte nativo a async com `AsyncAttrs, DeclarativeBase`
- `async_sessionmaker` com `expire_on_commit=False` para performance

**Alternatives considered**:
- `postgresql+psycopg_async` — mais novo, menos documentado
- Sync SQLAlchemy com threads — mais complexo, menos performante

### Decision: Pydantic Settings com Env File

**Decision**: Usar `BaseSettings` do pydantic-settings carregando de `.env`.

**Rationale**:
- Validação de tipos nas configurações
- Validações customizadas (ex: DATABASE_URL obrigatório)
- Integração nativa com FastAPI dependency injection

**Pattern**:
```python
class Settings(BaseSettings):
    model_config = SettingsConfigDict(env_file=".env")
    database_url: str  # obrigatório
    api_rate_limit: int = 60  # default
```

### Decision: Rate Limiting com slowapi + Fallback

**Decision**: Usar `slowapi` com backend Redis para rate limiting, com fallback para in-memory se Redis indisponível.

**Rationale**:
- Integração simples com FastAPI via middleware
- Redis já está no docker-compose para cache
- Suporta limit por IP customizável
- Fallback in-memory evita SPOF (Single Point of Failure)

### Decision: API Key Authentication (MVP)

**Decision**: Implementar autenticação simples via header `X-API-Key` para MVP.

**Rationale**:
- Requisito de segurança mínimo para proteger endpoints
- Simples de implementar e usar (uma env var)
- Suficiente para MVP de hackathon
- Futuro: evoluir para OAuth2/JWT

**Implementation**:
- Middleware valida `X-API-Key` header
- Exceções: `/health`, `/docs`, `/redoc` (endpoints públicos)
- Retorna 401 se inválido/ausente

## Data Model (Phase 1)

### Entity: Job

O modelo `Job` representa uma análise de modelagem de ameaças em execução.

```python
# src/models/job.py
from datetime import datetime
from uuid import UUID

from sqlalchemy import Enum, String, DateTime
from sqlalchemy.orm import Mapped, mapped_column

from src.models.base import Base


class JobStatus(str, Enum):
    PENDING = "pending"
    PROCESSING = "processing"
    COMPLETED = "completed"
    FAILED = "failed"


class Job(Base):
    """Análise job de modelagem de ameaças."""
    
    __tablename__ = "jobs"
    
    id: Mapped[UUID] = mapped_column(primary_key=True, default=uuid4)
    status: Mapped[JobStatus] = mapped_column(Enum(JobStatus), default=JobStatus.PENDING)
    input_image_path: Mapped[str] = mapped_column(String(500))
    output_report_path: Mapped[str | None] = mapped_column(String(500), nullable=True)
    error_message: Mapped[str | None] = mapped_column(String(1000), nullable=True)
    created_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), default=utc_now)
    updated_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True), 
        default=utc_now, 
        onupdate=utc_now
    )
```

### Relationships

- `Job` é independente (não tem foreign keys nesta spec)
- Será relacionado com `reports` nas specs futuras

### Validation Rules

- `input_image_path` deve ser caminho absoluto válido
- `status` só pode transicionar: PENDING → PROCESSING → (COMPLETED | FAILED)
- `output_report_path` obrigatório apenas quando status = COMPLETED
- `error_message` obrigatório apenas quando status = FAILED

from typing import Annotated
from fastapi import Depends

async def get_db() -> AsyncGenerator[AsyncSession, None]:
    async with async_session() as session:
        yield session

async def get_settings() -> Settings:
    return Settings()

async def get_storage() -> LocalFileStorage:
    return LocalFileStorage(settings.storage_path)

SessionDep = Annotated[AsyncSession, Depends(get_db)]

## Quickstart.md (Phase 1)

### Prerequisites

- Docker 24.x + Docker Compose
- Python 3.11+ (para desenvolvimento local)
- Poetry (opcional) ou pip

### Setup

```bash
# 1. Clonar e entrar no repo
git clone <repo>
cd hackaton-fiap-fase-5

# 2. Criar branch
git checkout -b feature/001-api-core

# 3. Configurar variáveis de ambiente
cp .env.example .env
# Editar .env com valores apropriados

# 4. Subir containers
docker-compose up --build

# 5. Verificar health
curl http://localhost:8000/health
```

### Validation Scenarios

#### Scenario 1: Health Check Success
```bash
# Dado: containers rodando
# Quando: chamar health endpoint
# Então: deve retornar status 200 com database connected

curl -s http://localhost:8000/health | jq .
# Expected: {"status": "healthy", "database": "connected", ...}
```

#### Scenario 2: Database Connectivity
```bash
# Dado: API rodando
# Quando: verificar logs
# Então: deve mostrar conexão com PostgreSQL bem-sucedida

docker-compose logs api | grep -i "database"
# Expected: "Database connection established"
```

#### Scenario 3: Rate Limiting Active
```bash
# Dado: API rodando com rate_limit = 60 req/min
# Quando: fazer 61 requisições em menos de 1 minuto
# Então: a 61ª deve retornar 429 Too Many Requests

for i in {1..61}; do curl -s -o /dev/null -w "%{http_code}\n" http://localhost:8000/health; done
# Expected: sequência de 200s seguida de 429
```

#### Scenario 4: Security Headers
```bash
# Dado: API rodando
# Quando: inspecionar headers de resposta
# Então: deve conter headers OWASP obrigatórios

curl -sI http://localhost:8000/health | grep -E "(X-Content-Type-Options|X-Frame-Options|Strict-Transport-Security)"
# Expected: todos os headers presentes
```

#### Scenario 5: Test Suite Pass
```bash
# Dado: ambiente configurado
# Quando: executar testes
# Então: todos devem passar com coverage >= 80%

docker-compose exec api pytest tests/ --cov=src --cov-report=term-missing
# Expected: coverage >= 80%, todos os testes passando
```
